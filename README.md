# rag-lab

Local-first RAG over your markdown docs. Point it at a folder (or a GitHub repo), ask
questions in plain language, get answers with citations — generated by a local model via
[Ollama](https://ollama.com). No API keys required to run.

**RAG that doesn't need OpenAI.** Every component (loader, chunker, embedder, retriever, LLM)
is swappable, and an eval harness lets you measure changes instead of guessing at them.

## Requirements

- Python 3.11+ and [uv](https://docs.astral.sh/uv/)
- A running [Ollama](https://ollama.com) with the default models pulled:

```bash
ollama pull nomic-embed-text   # embeddings
ollama pull llama3.2:3b        # answer generation
```

## Install

```bash
git clone https://github.com/aboydnw/local-llm-rag.git
cd local-llm-rag
uv sync
```

Commands below use `uv run rag-lab`; drop the `uv run` prefix if you install it on your PATH.

## Quickstart

```bash
uv run rag-lab config init                      # write a default rag.yml
uv run rag-lab ingest ./path/to/markdown        # build a local index (rag.db)
# ...or point straight at a GitHub repo:
uv run rag-lab ingest --repo developmentseed/titiler

uv run rag-lab ask "how do I add a custom raster source?"
```

`ask` retrieves the most relevant chunks (hybrid keyword + vector search), hands them to the
local model, and prints an answer followed by a numbered source list. Use `--show-chunks` to
see what was retrieved, and `rag-lab inspect chunks` to debug the index.

Configuration lives in `rag.yml` — swap the chunker, embedder, retriever weights, or LLM there.

## Evaluating quality

rag-lab ships with an eval harness so you can measure changes rather than guess at them.

```bash
rag-lab eval --golden golden.yml --report report.md
rag-lab eval --golden golden.yml --report report.md --previous eval-reports/baseline.md  # diff
rag-lab eval --golden golden.yml --report report.md --judge  # opt-in Anthropic LLM judge
```

The harness scores retrieval (recall@k, MRR) and answer quality (keyword coverage, plus an
optional model-graded judge). The `--judge` flag needs the optional `judge` extra
(`uv pip install 'rag-lab[judge]'`) and an `ANTHROPIC_API_KEY`; everything else runs fully local.

`examples/devseed-oss/` holds a golden set covering titiler, eoAPI, and stac-fastapi.
`eval-reports/` is the project's eval trail — every report is committed so you can see how a
change moved the metrics.
